# 上下文管理

## 作用
- 在“明确需要恢复或保存上下文”时，执行最小上下文闭环。

## 触发边界

### 触发
- 用户明确说“恢复会话/续做上次任务/读取上下文”。
- 用户明确说“保存当前状态/生成会话摘要”。

### 不触发
- 仅因仓库存在 `KCOS/` 不自动触发。
- 普通编码与排障过程不默认插入本技能。

## 协议引用
- 公共规则：`KCOS/protocol/p0-rules.md`
- 会话清单：`KCOS/protocol/ai-playbook.md`

## 最小流程
1. 读取 `KCOS/context/current-task.md`（若存在）。
2. 读取 `KCOS/context/session-latest.md`（若存在）。
3. 输出：当前任务、状态、待解决问题、下一步。
4. 需要保存时更新：
   - `KCOS/context/current-task.md`
   - `KCOS/context/session-{YYYYMMDD}-{n}.md`
   - `KCOS/context/session-latest.md`

## 输出契约
- 恢复时输出：`当前任务 + 当前状态 + 主要进展 + 待解决问题 + 本次目标`。
- 保存时输出：`本次产出摘要 + 关键决策 + 后续动作`。

## 执行规则
- 信息不明确先提澄清问题。
- 不编造历史上下文。
- 如新增/修改知识文档，结束前执行 `python3 KCOS/scripts/kcos_p0.py sync`。

## 手动触发示例
- "恢复会话"
- "读取上次进度"
- "保存当前状态"
